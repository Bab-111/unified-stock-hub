# 📈 Unified Stock Hub

A single GitHub Actions automation that consolidates three stock tools into one:
- **Screener** — scans 134 stocks across 8 criteria, finds breakout candidates
- **Monitor** — deep technical + fundamental analysis on your personal watchlist
- **News** — multi-source news aggregation per ticker + general market

Output: one self-contained HTML report, auto-deployed to GitHub Pages. Accessible from any device — desktop, laptop, or mobile.

> `https://[your-username].github.io/unified-stock-hub/`

Set up GitHub Pages (see below) and the link goes live after the first workflow run.

## Features

| Feature | Description |
|---|---|
| 📊 Breakout Screener | 134-stock universe, 8 weighted criteria (volume, MA200, MFI, institutional ownership, sector momentum, breakout pattern, IV, historical) |
| 👁 Watchlist Monitor | RSI, MACD, all MAs, fundamentals (PE/ROE/margins), options Greeks, earnings countdown, volume spikes |
| 📰 News | Yahoo Finance RSS + Google News + Finviz scraping + Reuters market feed, deduped |
| 🤖 LLM Supervision | Claude Haiku validates picks cross-module, flags contradictions, logs audit trail |
| ⚠️ Conflict Detector | Rule-based: warns when screener rates a stock bullish but monitor shows overbought RSI / bearish MACD |
| 🔍 Hallucination Guard | Flags any ticker in LLM output not present in actual run data |
| 🌏 Taiwan Timezone | All timestamps in both TW and ET. Watchlist supports `.TW` stocks |
| 📱 Mobile-First | Responsive layout, bottom nav on small screens, dark/light toggle |
| 💰 Free | yfinance + RSS feeds, no paid data APIs required. Only cost: optional Anthropic API key for LLM |

## Setup (5 minutes)

### 1. Create the repo

```bash
git clone https://github.com/[your-username]/unified-stock-hub
# or: create a new repo on GitHub and push these files
```

### 2. Enable GitHub Pages

Go to **Settings → Pages → Source** and set:
- **Branch:** `gh-pages`
- **Folder:** `/ (root)`

The `gh-pages` branch is created automatically on the first workflow run.

### 3. Add the API key (optional)

Go to **Settings → Secrets and variables → Actions → New repository secret**:
- **Name:** `ANTHROPIC_API_KEY`
- **Value:** your Anthropic API key

Without this, the report still runs fully — the LLM supervision panel just shows a notice.

### 4. Trigger the first run

Go to **Actions → Unified Stock Hub → Run workflow → Run workflow**

Select `full` for the first run. The report will be live at your Pages URL within ~4 minutes.

## Customization

### Watchlist tickers
Edit `config/watchlist.json` → `tickers` array. Supports US and Taiwan (`.TW`) symbols.

```json
{
  "tickers": ["AAPL", "TSLA", "2884.TW", "NVDA"]
}
```

### Screener universe
Edit `config/universe.csv` — one ticker per line under the `Symbol` header.

### Screener thresholds
Edit `config/config.json`:

```json
{
  "volume_spike_threshold": 2.0,
  "breakout_threshold": 1.03,
  "top_picks": 5,
  "inst_ownership_high": 60
}
```

### Schedule
Edit `.github/workflows/unified-stock-monitor.yml` — the `cron:` entries follow standard cron UTC syntax.

## LLM Supervision Details

The supervisor runs **after** all modules complete. It:

1. Builds a compact prompt (~200 tokens) summarizing screener picks + monitor alerts
2. Runs the conflict detector (rule-based, free) to find screener/monitor contradictions
3. Calls Claude Haiku with max 300 output tokens
4. Checks the response for hallucinated ticker symbols not in the actual data
5. Saves result to `output/llm_log.json` (last 10 entries shown in the LLM Log tab)

**Graceful degradation:** if `ANTHROPIC_API_KEY` is not set, all other features work normally.

## Troubleshooting

**Workflow fails at "Install dependencies"** — check `requirements.txt` and Python version (needs 3.11+)

**No data for a ticker** — yfinance occasionally has gaps; the module skips and logs the error without crashing

**GitHub Pages not updating** — check the `gh-pages` branch exists and Pages is pointing to it

**LLM note says "disabled"** — add `ANTHROPIC_API_KEY` as a repo secret (Settings → Secrets)
